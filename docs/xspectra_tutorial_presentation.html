물리 모델Core-hole approximation

흡수 edge 계산은 core electron을 비어 있는 상태로 올리는 과정이므로, 남은 core-hole이 주변 전자를 끌어당깁니다.

왜 근사가 필요한가

  • Ground-state DOS만 보면 core-hole이 만든 final-state potential을 빠뜨립니다.
  • Core-hole은 unoccupied state의 에너지와 모양을 바꾸고, 특히 onset 근처 피크를 움직일 수 있습니다.
  • FCH/HCH/no-hole 비교는 “흡수 뒤 screening을 얼마나 넣었는가”를 비교하는 실습입니다.
  • 이 근사는 완전한 excited-state 동역학이 아니라, 스펙트럼 해석을 위한 계산 가능한 final-state 모델입니다.
물리 모델 선택코어홀 근사

코어홀 모델은 final-state interaction을 바꿉니다

코어홀 없음

빠른 기준 계산입니다. 배우기에는 좋지만 강한 final-state effect를 놓칠 수 있습니다.

0

`tot_charge = 0`

전체 코어홀

흡수 원자에서 core electron 하나를 제거한 모델입니다. perturbation이 더 큽니다.

+1

`tot_charge = +1.0`

절반 코어홀

튜토리얼 비교와 screening을 위한 흔한 절충 모델입니다.

+0.5

`tot_charge = +0.5`

`filecore`는 core-hole이 없는 원래 pseudopotential에서 추출한 core wavefunction 파일을 가리킵니다.

Diamond 예제에서는 XSpectra 입력, 실행, plotting 흐름을 먼저 익힙니다.

공통 개념흡수 원자 species

흡수 원자는 보통 별도의 species 라벨로 지정합니다.

ATOMIC_SPECIES Sr 87.62 Sr...UPF Ti 47.867 Ti...UPF Oh 15.999 O...UPF ! 흡수 원자 O 15.999 O...UPF ATOMIC_POSITIONS crystal Sr 0.00 0.00 0.00 Ti 0.50 0.50 0.50 Oh 0.50 0.50 0.00

초보자가 자주 헷갈리는 점

XSpectra는 흡수 원자의 species 번호를 사용합니다. 단순히 “파일에서 세 번째 원자”라는 뜻이 아닙니다.

xiabs = 3 # Oh가 species 3
대표 예제SrTiO3 O K-edge

O K-edge: EELS에서 익숙하고 결합 해석이 비교적 직접적인 edge

해석 목표

  • O 1s → 비어 있는 O 2p-like 상태로의 전이를 봅니다.
  • SrTiO3에서는 O 2p가 Ti 3d 상태와 강하게 hybridize합니다.
  • 피크 모양은 국소 Ti–O 결합과 octahedral 전자구조를 논의하는 데 도움이 됩니다.
$ cd /scratch/$USER/xspectra-tutorial $ qsub scheduler/pbs_srtio3_okedge.pbs $ qstat -u "$USER" $ more xspectra-sto-ok-ch.o<jobid> # 작업 완료 뒤 로그인 노드에서 그림을 만듭니다 $ python3 scripts/make_presentation_plots.py $ ls docs/assets/presentation_srtio3_eels_comparison.png

`pbs_srtio3_okedge.pbs`는 O K-edge no-core-hole, FCH, HCH unit-cell 계산을 모두 제출합니다. 작업이 도는 동안 supercell과 EELS 정렬 슬라이드를 읽고, 완료 뒤 `scripts/make_presentation_plots.py`로 EELS 비교 그림을 갱신합니다.

물리 모델왜 큰 cell이 필요한가

Core-hole은 국소적인 final-state perturbation이므로, 주기적 복사본과의 상호작용을 줄여야 합니다.

해석 기준

  • 주기 경계에서는 core-hole도 주기적으로 반복되므로 cell이 작으면 hole-image 상호작용이 생깁니다.
  • 작은 unit cell 계산은 가까운 이웃 cell의 core-hole을 너무 강하게 느낄 수 있습니다.
  • Supercell은 이 인공 상호작용을 줄이기 위한 수렴성 점검입니다.
  • 피크 위치와 상대 세기가 cell 크기에 따라 크게 바뀌면 아직 수렴했다고 보기 어렵습니다.
비교 준비EELS preprocessing

실험 EELS를 넣을 때는 background 처리, energy calibration, normalization, rigid shift를 분리해서 생각합니다.

# 이 슬라이드 그림에 쓰는 실험 파일 $ head SrTiO3/O-K/data_blue_clean.csv Energy,Intensity 525.3271028037383,1093.75 525.4392523364486,1081.25 # 그림 생성: 실험 peak에 계산 peak를 rigid shift $ python3 scripts/make_presentation_plots.py $ ls docs/assets/presentation_srtio3_eels_comparison.png

스크립트가 하는 일

  • `SrTiO3/O-K/data_blue_clean.csv`의 첫 열은 energy loss, 둘째 열은 intensity입니다.
  • 실험 intensity를 normalize하고 `gaussian_filter1d`로 약하게 smooth합니다.
  • 각 곡선의 앞쪽 40% 구간에서 가장 큰 peak를 찾아 계산 energy 축 전체에 같은 shift를 더합니다.
  • 새 실험 데이터는 같은 CSV 형식으로 이 파일을 교체하거나, script 안의 입력 경로를 바꿉니다.
  • 정렬 뒤에는 절대 에너지보다 peak 순서, onset 근처 모양, unit-cell/supercell 차이를 봅니다.
비교실험 vs 계산

에너지 정렬 없이 절대 에너지를 맞추려 하지 말고, 모양과 경향부터 비교하세요.

반드시 기억할 점

  • 계산 스펙트럼은 상대적인 에너지 기준을 사용합니다.
  • 실험 EELS는 보통 background 처리와 calibration이 필요합니다.
  • 이 그림은 δ≈0 O K-edge 실험 곡선과 HCH reference 계산을 비교합니다.
  • 먼저 피크 순서, onset shift, core-hole 경향을 정성적으로 비교하세요.
  • XSpectra의 broadening은 실제 장비 응답 전체를 의미하지 않습니다.

Supercell 예제는 계산 자원과 병렬 설정을 먼저 확인한 뒤 실행합니다.

HCH supercell

SrTiO3 2x2x2 supercell은 원자 40개입니다. 흡수 원자 하나는 HCH UPF를 쓰고 `tot_charge = +0.5`를 유지합니다.

MPI 옵션

XSpectra는 k-point pool 병렬화로 `-nk 4`를 사용합니다. SCF 예제는 이 튜토리얼 환경에서 `-nd 1`로 diagonalization 실패를 피합니다.

스케줄러 전용

Supercell은 로그인 노드에서 실행하지 않습니다. 첫 실습에서는 제출하지 않고 `reference_output/`의 결과로 비교합니다.

문제 해결가장 먼저 확인할 체크리스트

`xiabs`가 단순한 원자 순서가 아니라 흡수 원자 species 라벨과 맞나요?

5. 출력

다시 실행하기 전에 `scf.out`과 `xspectra.out`의 마지막 80줄을 읽어 보세요.

6. 대안

대기열이 느리면 `reference_output/`을 사용해 물리 해석 논의를 계속하세요.

심화시간이 남으면

첫 시간에 너무 많이 넣지 마세요. 아래 내용은 심화 주제로 남겨 둡니다.

Supercell

인위적인 주기적 core-hole 상호작용을 줄일 수 있지만 계산 비용이 커집니다. unit-cell 계산을 이해한 뒤 다루면 좋습니다.

Convergence

cutoff, k-point, band 수, broadening, core-hole 모델, supercell 크기는 모두 스펙트럼을 바꿀 수 있습니다.

정량 EELS

정밀한 에너지 정렬, 장비 broadening, geometry, 두께, background, 그리고 경우에 따라 beyond-DFT 효과가 필요합니다.

벤치마킹

MPI scaling 스크립트는 심화 사용자에게 유용하지만, 처음 리눅스를 배우는 실습에서는 우선순위가 낮습니다.

정리여러분이 기억할 것

첫 XSpectra 실습에서는 네 가지면 충분합니다.

1

작업 흐름

UPF → core wavefunction → SCF → XSpectra → spectrum.

2

흡수 원자

별도의 species 라벨과 올바른 `xiabs`를 사용합니다.

3

클러스터

로그인 노드에서는 설정을 확인하고, 계산은 스케줄러로 실행합니다.

4

해석

먼저 정성적 경향을 비교하고, 에너지 정렬은 나중에 신중히 합니다.

이 네 가지를 설명할 수 있으면 오늘 실습의 핵심을 이해한 것입니다.

진행 확인각 단계에서 점검

실습은 각 슬라이드의 명령을 따라가며 진행합니다.

$ ./check_setup.sh $ qstat -u "$USER" # 작업이 qstat에서 사라지고 출력 파일이 생기면 읽습니다 $ tail -80 diamond/diamond.xspectra.out $ tail -80 SrTiO3/O_Kedge/xspectra.out $ tail -80 SrTiO3/O_Kedge_HCH/xspectra.out # 작업이 아직 대기 중이면 reference_output으로 그림 비교를 계속합니다 $ python3 scripts/make_presentation_plots.py $ ls docs/assets/presentation_srtio3_eels_comparison.png

계산

작업 제출과 대기열 확인은 해당 실습 슬라이드에서 바로 수행합니다.

출력

그림을 보기 전에 `scf.out`과 `xspectra.out`의 마지막 부분을 확인합니다.

대안
