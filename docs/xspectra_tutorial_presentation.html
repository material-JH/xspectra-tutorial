XSpectra 튜토리얼학생용 실습 가이드

EELS 직관에서 제일원리 스펙트럼까지

Quantum ESPRESSO + XSpectra를 사용하는 슈퍼컴퓨터 실습입니다. EELS 기본 개념은 이미 배웠다고 가정하고, 오늘은 작업 흐름, 결과 해석, 안전한 클러스터 사용에 집중합니다.

QE → XSpectra → EELS 비교

여러분은 작은 예제를 직접 실행하고, 입력/출력 파일을 확인하며, 스펙트럼 특징을 비어 있는 전자 상태와 연결해 보게 됩니다.

준비이 실습에서 하는 것 / 하지 않는 것

첫 계산을 “마법”이 아니라 이해 가능한 절차로 만드는 것이 목표입니다.

대상

TEM/EELS를 사용하는 대학원생과 포닥을 위한 자료입니다. 리눅스, 클러스터, QE 입력 파일, XSpectra가 처음이어도 괜찮습니다.

가정

EELS의 기본 개념은 이 실습 전에 배웠다고 가정합니다. 여기서는 edge onset, background subtraction, detector geometry를 다시 자세히 다루지 않습니다.

오늘의 목표

XSpectra 전체 흐름을 직접 실행하고 출력 파일을 읽은 뒤, SrTiO3 EELS와 정성적인 스펙트럼 경향을 비교합니다.

배경목적

EELS 소개 뒤에 왜 XSpectra를 사용할까요?

실험과 전자구조를 연결하기

  • Edge는 core level에서 비어 있는 상태로의 전이를 봅니다.
  • 피크 모양은 국소 결합, 대칭성, 산화 상태, hybridization을 반영합니다.
  • DFT는 이런 특징을 해석하기 위한 통제된 모델을 제공합니다.

처음부터 기대치를 분명히 하기

  • 이 실습은 완전한 정량 EELS 시뮬레이션이 아닙니다.
  • 에너지 정렬, broadening, 시편 두께, 실험 geometry는 별도의 주의가 필요합니다.
  • 처음에는 작업 흐름을 익히고 정성적으로 해석하는 것이 가장 중요합니다.
개념 지도각 코드의 역할

QE는 전자구조를 구하고, XSpectra는 edge 응답을 계산합니다.

구조 + pseudopotential

원자, 격자, 흡수 원자 라벨, UPF 파일, core-hole 선택을 정합니다.

→

pw.x SCF

`tmp/`에 charge density와 wavefunction 데이터를 저장합니다.

→

xspectra.x

core wavefunction과 SCF 데이터를 사용해 `xanes.dat`를 만듭니다.

매개변수를 바꾸기 전에 이 흐름을 먼저 머릿속에 넣어 두세요.

클러스터 사용계산을 시작하기 전에

슈퍼컴퓨터에서 가장 중요한 규칙

로그인 노드

  • 파일 편집
  • `git`, `more`, `vim` 실행
  • `bash check_setup.sh` 실행
  • 작은 출력 파일 확인

계산 노드 / 스케줄러

  • `pw.x` 실행
  • `xspectra.x` 실행
  • MPI rank 사용
  • SrTiO3와 supercell 예제 실행

큰 MPI 계산은 로그인 노드에서 실행하지 마세요. 이 환경에서는 PBS `qsub`/`qstat` 또는 대화형 계산 노드를 사용하세요.

저장소여러분이 보게 될 파일

몇 개의 파일만 이해하면 전체 실습 흐름을 따라갈 수 있습니다

env.sh.example
먼저 클러스터 module 줄만 맞춥니다. `pw.x`, `xspectra.x`, `BIN_DIR`, `QE_ROOT`는 가능한 경우 자동 감지됩니다.
check_setup.sh
가벼운 사전 점검 스크립트입니다. QE 실행 파일, pseudopotential, Python package, launcher를 확인합니다.
scheduler/
PBS/Slurm 작업 제출 템플릿입니다. 로그인 노드에서 실수로 계산하지 않도록 도와줍니다.
SrTiO3/
EELS와 연결되는 대표 예제입니다: O K-edge, Ti L-edge, core-hole 변형, plot.
사전 준비source tree와 pseudopotential

대부분의 학생은 QE를 직접 clone하지 않고 시작합니다.

UPF는 이미 포함

Diamond C K-edge pseudo는 `diamond/pseudo/`에 있고, SrTiO3 예제 pseudo는 `SrTiO3/pseudo/`에 있습니다.

실행 파일은 module 우선

클러스터 module을 load한 뒤 `source env.sh`를 실행하면 `pw.x`, `xspectra.x`, `ld1.x` 경로를 먼저 자동으로 찾습니다.

`~/q-e`는 필요할 때만

`upf2plotcore.sh`가 module 설치에 없을 때만 QE source tree를 `~/q-e` 등에 받습니다. 이 tutorial repo 안에 `q-e/`를 clone하거나 commit하지 마세요.

실습첫 명령

모두 같은 환경에서 시작하세요

$ cd /scratch/$USER $ git clone https://github.com/material-JH/xspectra-tutorial.git $ cd xspectra-tutorial $ cp env.sh.example env.sh $ vim env.sh # 먼저 module load 줄만 맞추세요 $ source env.sh # QE 경로 자동 감지와 경고 확인 # TOOLS_DIR는 upf2plotcore.sh 경고가 날 때만 직접 지정합니다 $ bash check_setup.sh

여기서 실패하면 물리 해석으로 넘어가지 말고 설정부터 고치세요. 초보자의 대부분 실패는 경로, module, 또는 누락된 `upf2plotcore.sh`에서 나옵니다.

스케줄러PBS 예시

튜토리얼 계산은 스케줄러를 통해 실행합니다

$ cd /scratch/$USER/xspectra-tutorial $ qsub scheduler/pbs_diamond.pbs $ qstat -u "$USER" $ more xspectra-diamond.o<jobid>

이 스크립트가 하는 일

  1. `env.sh`를 불러옵니다
  2. `norm_skl` queue에 작업을 제출합니다
  3. 튜토리얼 기본값인 4 MPI rank를 사용합니다
  4. 먼저 Diamond 예제만 실행합니다

Slurm 클러스터에서는 `scheduler/slurm_diamond.sbatch`와 `sbatch`/`squeue` 흐름으로 바꿉니다.

예제 1Diamond C K-edge

Diamond는 재료과학 결론을 내기 위한 예제가 아니라, 계산 흐름을 익히기 위한 예제입니다.

확인할 점

  • `C.wfc`가 UPF pseudopotential에서 만들어집니다.
  • `pw.x`가 저장된 charge-density 데이터를 만듭니다.
  • `xspectra.x`가 `xanes.dat`를 만듭니다.
  • core-hole과 no-core-hole 스펙트럼은 서로 다릅니다.
공통 개념흡수 원자 species

흡수 원자는 보통 별도의 species 라벨로 지정합니다.

ATOMIC_SPECIES Sr 87.62 Sr...UPF Ti 47.867 Ti...UPF Oh 15.999 O...UPF ! 흡수 원자 O 15.999 O...UPF ATOMIC_POSITIONS crystal Sr 0.00 0.00 0.00 Ti 0.50 0.50 0.50 Oh 0.50 0.50 0.00

초보자가 자주 헷갈리는 점

XSpectra는 흡수 원자의 species 번호를 사용합니다. 단순히 “파일에서 세 번째 원자”라는 뜻이 아닙니다.

xiabs = 3 # Oh가 species 3
대표 예제SrTiO3 O K-edge

O K-edge: EELS에서 익숙하고 결합 해석이 비교적 직접적인 edge

해석 목표

  • O 1s → 비어 있는 O 2p-like 상태로의 전이를 봅니다.
  • SrTiO3에서는 O 2p가 Ti 3d 상태와 강하게 hybridize합니다.
  • 피크 모양은 국소 Ti–O 결합과 octahedral 전자구조를 논의하는 데 도움이 됩니다.
$ cd SrTiO3/O_Kedge $ bash run.sh $ cd ../.. $ cd SrTiO3 $ python3 plot_spectra.py
물리 모델 선택코어홀 근사

코어홀 모델은 final-state interaction을 바꿉니다

코어홀 없음

빠른 기준 계산입니다. 배우기에는 좋지만 강한 final-state effect를 놓칠 수 있습니다.

0

`tot_charge = 0`

전체 코어홀

흡수 원자에서 core electron 하나를 제거한 모델입니다. perturbation이 더 큽니다.

+1

`tot_charge = +1.0`

절반 코어홀

튜토리얼 비교와 screening을 위한 흔한 절충 모델입니다.

+0.5

`tot_charge = +0.5`

중요: `filecore` wavefunction은 원래의 non-core-hole pseudopotential에서 추출합니다.

비교실험 vs 계산

에너지 정렬 없이 절대 에너지를 맞추려 하지 말고, 모양과 경향부터 비교하세요.

반드시 기억할 점

  • 계산 스펙트럼은 상대적인 에너지 기준을 사용합니다.
  • 실험 EELS는 보통 background 처리와 calibration이 필요합니다.
  • 먼저 피크 순서, onset shift, core-hole 경향을 정성적으로 비교하세요.
  • XSpectra의 broadening은 실제 장비 응답 전체를 의미하지 않습니다.
두 번째 edgeSrTiO3 Ti L-edge

Ti L-edge는 EELS 사용자에게 직관적이지만 조심해서 해석해야 합니다.

이 예제에서 볼 점

  • Ti 2p → Ti 3d-like 비어 있는 상태 전이를 봅니다.
  • valence, white-line shape, crystal-field 표현과 연결합니다.
  • 흡수 원자 species 라벨을 다시 확인합니다: `Tih`, `xiabs = 2`.

중요한 한계

  • 튜토리얼 입력은 `edge='L2'`처럼 하나의 edge 설정만 사용할 수 있습니다.
  • 실제 EELS L2,3 해석에는 spin-orbit, multiplet, geometry, broadening을 더 신중히 다뤄야 합니다.
  • 이 초보자 실습에서는 정성적으로만 사용하세요.
출력성공했을 때 보이는 것

그림만 기다리지 말고 출력 파일을 먼저 읽으세요.

$ cd SrTiO3/O_Kedge $ more scf.out $ tail -80 xspectra.out $ ls *.dat $ cd ../.. # repo root로 돌아와 reference와 비교 $ python3 scripts/compare_reference.py \ SrTiO3/O_Kedge/O_Kedge.dat \ reference_output/SrTiO3/O_Kedge/O_Kedge.dat

생성되어야 할 파일

  • `scf.out`: QE SCF 로그
  • `xspectra.out`: XSpectra 로그
  • `O.wfc` / `Ti.wfc`: 추출된 core wavefunction
  • `O_Kedge.dat`, `Ti_Ledge.dat`: 스펙트럼
  • `tmp/`: scratch 데이터이며 보통 보관하지 않습니다
심화 실습docs/09_supercell_and_mpi.md

Supercell 예제는 계산 자원과 병렬 설정을 먼저 확인한 뒤 실행합니다.

HCH supercell

SrTiO3 2x2x2 supercell은 원자 40개입니다. 흡수 원자 하나는 HCH UPF를 쓰고 `tot_charge = +0.5`를 유지합니다.

MPI 옵션

XSpectra는 k-point pool 병렬화로 `-nk 4`를 사용합니다. SCF 예제는 이 튜토리얼 환경에서 `-nd 1`로 diagonalization 실패를 피합니다.

스케줄러 전용

Supercell은 로그인 노드에서 실행하지 않습니다. 이 환경에서는 PBS `qsub` 템플릿을 기준으로 조정하고, Slurm 클러스터에서는 Slurm 템플릿으로 바꿉니다.

문제 해결가장 먼저 확인할 체크리스트

작업이 실패하면 지루하지만 확실한 체크리스트를 따라가세요.

1. 환경

`bash check_setup.sh`가 통과했나요? `pw.x`, `xspectra.x`, `upf2plotcore.sh`가 의도한 QE build에서 온 것인가요?

2. 파일

QE가 모든 UPF 파일을 열 수 있나요? `filecore='O.wfc'` 또는 `Ti.wfc` 파일이 실제로 있나요?

3. 흡수 원자

`xiabs`가 단순한 원자 순서가 아니라 흡수 원자 species 라벨과 맞나요?

4. 스케줄러

충분한 wall time과 올바른 MPI launcher로 제출했나요?

5. 출력

다시 실행하기 전에 `scf.out`과 `xspectra.out`의 마지막 80줄을 읽어 보세요.

6. 대안

대기열이 느리면 `reference_output/`을 사용해 물리 해석 논의를 계속하세요.

심화시간이 남으면

첫 시간에 너무 많이 넣지 마세요. 아래 내용은 심화 주제로 남겨 둡니다.

Supercell

인위적인 주기적 core-hole 상호작용을 줄일 수 있지만 계산 비용이 커집니다. unit-cell 계산을 이해한 뒤 다루면 좋습니다.

Convergence

cutoff, k-point, band 수, broadening, core-hole 모델, supercell 크기는 모두 스펙트럼을 바꿀 수 있습니다.

정량 EELS

정밀한 에너지 정렬, 장비 broadening, geometry, 두께, background, 그리고 경우에 따라 beyond-DFT 효과가 필요합니다.

벤치마킹

MPI scaling 스크립트는 심화 사용자에게 유용하지만, 처음 리눅스를 배우는 실습에서는 우선순위가 낮습니다.

정리여러분이 기억할 것

첫 XSpectra 실습에서는 네 가지면 충분합니다.

1

작업 흐름

UPF → core wavefunction → SCF → XSpectra → spectrum.

2

흡수 원자

별도의 species 라벨과 올바른 `xiabs`를 사용합니다.

3

클러스터

로그인 노드에서는 설정을 확인하고, 계산은 스케줄러로 실행합니다.

4

해석

먼저 정성적 경향을 비교하고, 에너지 정렬은 나중에 신중히 합니다.

이 네 가지를 설명할 수 있으면 오늘 실습의 핵심을 이해한 것입니다.
